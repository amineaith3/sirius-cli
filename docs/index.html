Quick Start Workflows

Scaffold a full system by passing one or more CSV files. Sirius will infer relationships if it sees table_id columns.

sirius-init init store_system \
  --csv users.csv \
  --csv orders.csv \
  --theme violet

Target production databases directly while reading from Excel. This sets up Postgres drivers in the generated project.

sirius-init init analytics \
  --excel data.xlsx \
  --pg \
  --theme emerald

Point Sirius at an existing SQLite file. It will reverse-engineer the schema via PRAGMAs and build the REST API around it.

sirius-init init legacy_app --db old_database.db

Generate a secure project with JWT authentication. Automatically creates Login screens and protects API routes.

sirius-init init secure_admin \
  --config schema.json \
  --auth \
  --admin-user admin \
  --admin-pass super_secret

CLI Reference

Flag Description
--csv <path> Path to input CSV file. Can be declared multiple times.
--excel <path> Path to input Excel (.xlsx) file. Can be declared multiple times.
--db <path> Path to a SQLite database to introspect.
--config, -c Path to JSON configuration defining explicit schema.
--theme, -t Frontend Tailwind theme (blue, indigo, emerald, amber, rose, sky, violet). Default: blue.
--pg, --mysql Generate project tailored for PostgreSQL or MySQL databases instead of SQLite.
--auth Generate JWT Bearer token authentication and Login UI.
--no-seed Skip automatically inserting rows from source files into the generated DB.
